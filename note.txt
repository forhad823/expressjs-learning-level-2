##. How ETag Caching Works in Express status: 200 ok, 304 not modified   
When you send a response in Express using functions like  res.json()  or       
  res.send() :                                                   
  1. Express automatically hashes the response body (your JSON object in         
  server.ts) and generates a unique identifier called an ETag (Entity Tag). 
  2. It sends this  ETag  value to the browser inside the response header.       
  3. The next time the browser requests  / , it sends the  If-None-Match  header 
  containing that ETag.                                                          
  4. Express intercepts the request, generates a new ETag hash for the current   
  response, and compares it to the browser's  If-None-Match  header.             
  5. Since the content is exactly the same, they match. Express overrides your   
  status code of  200  and returns a  304 Not Modified  response with an empty   
  body to save network bandwidth.
